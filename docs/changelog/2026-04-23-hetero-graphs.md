# 2026-04-23 — Heterogeneous Graphs + Multi-Graph Demos + CSV Cleanup

## Added
- **Demo Excel datasets** (generated by `backend/scripts/generate_excel_demos.py`):
  - `demo_multigraph_homo.xlsx` — 10 graphs, single node/edge type, `graph_regression` target `target_delay`
  - `demo_multigraph_hetero.xlsx` — 10 graphs, 3 node types (cell/pin/net), 3 edge types (cell2pin/pin2pin/pin2net), `graph_regression` target `total_wirelength`
- **Heterogeneous ingestion**: `parse_excel_file()` now returns `node_dfs`, `edge_dfs`, `canonical_edges`, and `is_heterogeneous`. Multi-Type per Level is accepted.
- **HeteroData conversion** (`backend/app/data/pyg_converter_hetero.py`): builds one `HeteroData` per graph with per-type `x` / `edge_index` tensors and graph-level `y`.
- **Heterogeneous GNN** (`backend/app/models/hetero_wrapper.py`): `HeteroGraphRegressor` wraps a homo backbone (GCN / GAT / SAGE) via `torch_geometric.nn.to_hetero`, pools per-type, and feeds a linear head.
- **Graph-list training**: `pyg_converter.py::dataframes_to_graph_list` produces one `Data` per graph for homogeneous graph-level tasks.
- **Pipeline branch**: `pipeline.run_training_task` now dispatches to hetero / homo-graph-list / node-single-Data paths based on `task_type` + `is_heterogeneous`.
- **Explore multi-graph stats** (backend + frontend): `graph_count`, `avg_nodes_per_graph`, `avg_edges_per_graph`, `is_heterogeneous`, `node_types`, `edge_types`, `canonical_edges`.
- **GraphPreview hetero colouring**: nodes are coloured by `node_type` (10-colour palette) when heterogeneous; legend shows node + edge types.
- **New endpoints**: `GET /demo-excels`, `GET /demo-excel/{id}`, `POST /{project_id}/load-demo-excel`.
- **Demo Excel card** on the Upload page with one-click Load / Download buttons.

## Changed
- `DatasetSummary`, `GenericExploreData`, `Report` gain hetero & multi-graph fields.
- Default task type is now `graph_regression` (previously `node_classification`).
- `pyg_converter.py` keeps only `dataframes_to_pyg_dynamic` (node-level) and `dataframes_to_graph_list` (new).

## Removed
- **CSV ingress** — legacy `/upload` (CSV), `/upload-folder`, `/load-demo`, `/sample-data`, CSV `evaluate-demo` / `evaluate` endpoints.
- **CSV data modules** — `app/data/ingestion.py`, `app/data/mock_loader.py`.
- **CSV demo datasets** — `backend/mock_data/` (5 demo directories + 4 root CSVs).
- **CSV generator** — `backend/generate_mock_data.py`.
- **CSV router** — `app/routers/datasets.py` (and its inclusion in `main.py`).
- **CSV-specific config** — `MOCK_DATA_DIR`, `NUMERIC_FEATURES`, `CELL_TYPES`, `LABEL_COLUMN`, `REGRESSION_LABEL` from `core/config.py`.
- **Legacy explore stats** — `compute_explore_stats` (circuit-specific) from `feature_engineering.py`; `ExploreData` pydantic schema.
- **Frontend legacy APIs** — `uploadProjectFolder`, `uploadProjectData`, `loadDemoData`, `downloadSampleData`, `listDemoDatasets`, `evaluateModelWithDemo`, `evaluateModelWithData`, `DemoDatasetInfo`.
- **Frontend Jest** — `api.test.ts`, `api-extended.test.ts` (all covered CSV-only APIs); replaced by `api-excel.test.ts` with full Excel coverage.
- **Upload page legacy UI** — folder picker, demo CSV datasets, Detected Graphs panel.
- **Models page evaluate UI** — CSV file upload + demo evaluation (can be re-added later for Excel-based evaluation).
- **Tests** — `tests/test_api.py`, `tests/test_ingestion.py`, `tests/test_pyg_converter.py` (all CSV-bound).

## Migration Impact
- Any external client relying on `/upload`, `/upload-folder`, `/load-demo`, `/sample-data`, `/datasets` (list + upload) — **breaking**. Replace with `/upload-excel`, `/load-demo-excel`, `/sample-excel`.
- `DatasetSummary` has new optional fields; existing parsers are unaffected.
